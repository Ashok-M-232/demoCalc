1. What is Networking?

Networking is the practice of connecting two or more computing devices (such as computers, servers, smartphones, routers, switches, and IoT devices) so that they can communicate and share resources like data, files, internet, printers, applications, and services.

A computer network enables:

  • Communication between users and systems
  • Centralized storage and resource sharing
  • Remote access capabilities
  • Efficient data transfer
  • Secure and controlled access to information

Note: Networking is the backbone of modern IT infrastructure, enterprise operations, cloud computing, cybersecurity, and SOC operations.

2. What is Computer Networking?

Computer networking refers to the architecture, design, configuration, management, and security of interconnected computing devices. It involves both hardware components and software configurations that enable devices to communicate using defined rules called protocols.

A computer network can be defined professionally in two ways:

  1. A collection of interconnected computing devices that can communicate and share resources.
  2. A system in which multiple computers are connected through wired or wireless channels to exchange data efficiently and securely.

3. Types of Networking

A. Hardware Networking

Deals with all physical components required to build and maintain a network.

  • Routers
  • Switches
  • Hubs
  • Repeaters
  • Cables (Ethernet, Fiber)
  • Network Interface Cards
  • Access Points

Purpose: Ensures physical connectivity and reliable data transmission.

B. Software Networking

Deals with software configurations, protocols, network operating systems, virtualization, and logically managing traffic.

  • Network Operating Systems
  • Routing configurations
  • Virtual LANs (VLAN)
  • Software Defined Networking (SDN)
  • Network security policies
  • Firewall configurations
  • VPN configurations

Purpose: Ensures logical control, efficient communication, and secure access across the network.

4. Types of Networks (Based on Coverage Area)

1. PAN ‚Äì Personal Area Network

A PAN is the smallest network designed for personal device connectivity within a very short range (a few meters).

  • Coverage: 1‚Äì10 meters typically
  • Use: Personal communication
  • Cost: Low and simple setup

Technologies: Bluetooth, Infrared, NFC, USB connections, Hotspot-based personal networks

Common examples: Mobile to Bluetooth headphones, Smartwatch connected to mobile, Laptop connected to mobile hotspot

2. LAN ‚Äì Local Area Network

A LAN connects multiple computers within a limited area such as homes, offices, schools, or corporate buildings.

  • Coverage: Up to 1‚Äì2 km depending on infrastructure
  • Speed: Very high (100 Mbps ‚Äì multiple Gbps)
  • Ownership: Controlled by a single organization
  • Security: Highly secure and manageable

Technologies: Ethernet, Wi-Fi

Examples: Office network, Home Wi-Fi network, College campus building network

3. MAN ‚Äì Metropolitan Area Network

A MAN connects multiple LANs within a city or metropolitan region, usually managed by ISPs or Telecom providers.

  • Coverage: 5‚Äì50+ km
  • Use: City-wide or large region networks
  • Ownership: Not owned by a single organization
  • Speed: Moderate to high, depends on infrastructure

Technologies: Fiber optic, Microwave links, Copper leased lines

Examples: City-wide corporate networks, Government departmental networks, University city-wide networks

4. WAN ‚Äì Wide Area Network

WAN connects multiple LANs and MANs over extremely large geographical areas, often across countries and continents.

  • Coverage: Countries and entire globe
  • Speed: Varies based on connection type
  • Architecture: Highly complex, managed by multiple providers
  • Cost: Expensive to deploy but essential for global operations

Technologies: Satellite links, Optical fiber, Undersea cables, MPLS, ISP backbone networks

Examples: The Internet, Global bank networking systems, Corporate global networks

5. Networking Topologies

Networking Topology refers to the physical or logical arrangement of network devices and how data travels between them. It defines communication structure, network performance, and fault tolerance capability.

Main Types of Topologies: Bus, Star, Ring, Mesh, Hybrid


Bus Topology

In Bus Topology, all devices are connected to a single main communication cable called the Bus or Backbone.

How it Works:

  • All devices share one main cable
  • Data travels along the single backbone
  • Each device checks if data belongs to it

Advantages:

  • Simple design
  • Low cost (fewer cables)
  • Easy to install

Disadvantages:

  • If backbone fails ‚Üí entire network stops
  • Performance decreases with more devices
  • Difficult to troubleshoot

Used in: Small networks, early LAN systems


Star Topology

In Star Topology, all devices are individually connected to a central device such as a Switch or Hub.

How it Works:

  • Each device has a dedicated connection to central device
  • All communication passes through the Switch/Hub

Advantages:

  • Highly reliable
  • Easy to manage & troubleshoot
  • Most widely used today

Disadvantages:

  • If central device fails ‚Üí network stops
  • Requires more cabling

Used in: Offices, Schools, Corporate LANs


Ring Topology

Devices are connected in a circular path where each device connects to exactly two others.

How it Works:

  • Data travels in one or both directions in a ring
  • Each device forwards data to the next

Advantages:

  • Better performance than Bus under high load

Disadvantages:

  • Single failure can break entire network
  • Difficult to troubleshoot
  • Adding/removing devices disrupts network

Used in: Older networks, Industrial networks


Mesh Topology

In Mesh topology, every device is connected to multiple devices.

Types:

  • Full Mesh ‚Äì Every device connects to every device
  • Partial Mesh ‚Äì Only important devices have multiple connections

Advantages:

  • Extremely reliable
  • Multiple communication paths
  • Excellent redundancy

Disadvantages:

  • Very expensive
  • Complex installation & management

Used in: Military, Data Centers, Secure Networks


Hybrid Topology

Hybrid Topology is a combination of two or more topologies such as Star + Bus or Star + Ring.

Advantages:

  • Highly flexible
  • Scalable and customizable
  • Suitable for large organizations

Disadvantages:

  • Expensive
  • Requires expert management

Used in: Universities, Large Enterprises, Complex Networks

  • Star ‚Üí All devices connect to central switch
  • Ring ‚Üí Circular communication loop
  • Mesh ‚Üí Every device connects to many devices

6. Networking Devices

Networking devices are hardware components that connect computers and other devices in a network, enabling communication, data sharing, internet connectivity, and secure operations.

Main Networking Devices: Switch, Router, Hub, Repeater, Access Point

Switch

A Switch is one of the most important networking devices used in modern LANs. It connects multiple devices like computers, printers, and servers within the same network.

How it Works:

  • Receives data from one device
  • Forwards it only to the intended device
  • Uses MAC addresses to identify devices

Why it is Important:

  • Reduces unnecessary traffic
  • Improves speed & performance
  • More secure than hubs

Used in: Offices, Companies, Schools, Data Centers

Simple Meaning: Switch = Smart device for internal communication

Router

A Router connects different networks together. Most commonly, it connects a Local Area Network (LAN) to the Internet.

How it Works:

  • Receives data from one network
  • Reads destination IP address
  • Finds best path and forwards data

Why it is Important:

  • Connects LAN to Internet
  • Assigns IP addresses
  • Supports NAT, Firewall & Security

Used in: Homes, Offices, Enterprises, ISPs

Simple Meaning: Router = Connects different networks & directs data

Hub

A Hub is an older networking device used to connect multiple devices, but unlike a switch, it is not intelligent.

How it Works:

  • Broadcasts data to all connected devices
  • Only the intended device uses it

Why it is Weak:

  • Creates unnecessary traffic
  • Less secure
  • Poor performance

Note: Rarely used today. Replaced by Switches.

Simple Meaning: Hub = Old & dumb broadcasting device

Repeater

A Repeater is used to extend the range of a network by strengthening weak signals.

Why Needed:

  • Signals weaken over long distances
  • Weak signals may cause data loss

How it Works:

  • Receives weak signal
  • Amplifies it
  • Re-transmits as strong signal

Used in: Large Buildings, Long Distance Networks, Wireless Coverage Areas

Simple Meaning: Repeater = Boosts weak network signals

Access Point (AP)

An Access Point provides wireless network connectivity (Wi-Fi). It allows wireless devices like mobiles, laptops, tablets to connect to a network.

How it Works:

  • Connects to router or switch
  • Converts wired network to wireless
  • Allows multiple users to connect

Why Important:

  • Provides Wi-Fi
  • Supports many users
  • Expands coverage

Used in: Offices, Colleges, Airports, Hotels, Public Wi-Fi

Simple Meaning: Access Point = Device that provides wireless connectivity

‚úî Simple Memory Summary

  • Switch ‚Üí Connects devices inside LAN, forwards data smartly
  • Router ‚Üí Connects different networks & internet
  • Hub ‚Üí Old broadcast device (not smart)
  • Repeater ‚Üí Boosts weak signals
  • Access Point ‚Üí Provides Wi-Fi connectivity

7. Important Networking Concepts

What is IP?

An IP (Internet Protocol) address is a unique numerical identifier assigned to every device connected to a network. It enables devices to send and receive data across networks.

Two versions:

  • IPv4 ‚Äì Example: 192.168.1.10
  • IPv6 ‚Äì Example: 2001:db8:85a3::8a2e

What is an IP Address?

An IP (Internet Protocol) Address is a unique numerical address assigned to every device connected to a network, such as computers, mobile phones, servers, routers, CCTV devices, and IoT gadgets.

Think of it like:

  • Your home has a house number
  • Your phone has a mobile number
  • Similarly, every device on a network has an IP address

This address helps devices identify each other and communicate over a network or the internet.

Why is IP Important?

Without an IP address:

  • Devices cannot find each other
  • Data cannot be delivered to the correct device
  • Internet communication is impossible

So IP works like a digital address system, ensuring that data goes to the right destination.

What Does IP Do?

An IP address:

  • Uniquely identifies each device
  • Helps locate devices on a network
  • Allows sending and receiving data correctly

Example:

  • If you search something on Google,
  • Your device sends a request using its IP address
  • Google responds back to that IP address
  • You receive the webpage

So IP is responsible for communication, tracking, and addressing.

Types of IP Versions

Currently, two IP versions are used:

1. IPv4 (Internet Protocol Version 4)

  • Oldest and most widely used version
  • Uses 32-bit addressing
  • Written in decimal format
  • Separated by dots
  • Example: 192.168.1.10
  • Total possible addresses: Around 4.3 billion
  • Since the internet grew massively, these addresses are running out.

2. IPv6 (Internet Protocol Version 6)

  • New and advanced version
  • Created to replace IPv4
  • Uses 128-bit addressing
  • Written in hexadecimal format
  • Separated by colons
  • Example: 2001:db8:85a3::8a2e
  • Provides extremely large number of addresses (almost unlimited), faster performance, and better security.

Simple Short Summary

  • IP Address = Unique number assigned to every device
  • Purpose = Identify device + Enable communication
  • Versions = IPv4 and IPv6

What is Private IP?

A private IP address is used inside private networks such as homes, offices, and organizations. It is not routable on the internet and is used only within internal networks.

Example ranges:

  • 10.0.0.0 to 10.255.255.255
  • 172.16.0.0 to 172.31.255.255
  • 192.168.0.0 to 192.168.255.255

Used in:

  • LAN networks
  • Home networks
  • Corporate internal environments

What is a Public IP Address?

A Public IP Address is a globally unique address assigned to your network by your Internet Service Provider (ISP). It is the identity of your network on the internet. Any device or network that needs to communicate over the internet must use a public IP.

You can think of it like your home address in the real world. Just as someone needs your home address to send you a letter, other networks need your public IP to communicate with you over the internet.

Key Characteristics:

  • It is globally unique (no two public IPs are the same at the same time).
  • It is assigned by ISP to your modem or router.
  • It is required for internet communication.
  • It is accessible over the internet, meaning external networks can reach it.
  • It can be Static (fixed) or Dynamic (changes automatically).

Example:

  • 8.8.8.8 ‚Üí Google Public DNS
  • 1.1.1.1 ‚Üí Cloudflare DNS

Where it is used:

  • Home internet connections
  • Company networks
  • Servers and websites
  • Cloud services

What is a MAC Address?

A MAC Address (Media Access Control Address) is a unique physical hardware address assigned to a device‚Äôs Network Interface Card (NIC) by the manufacturer.

It works at the Data Link Layer (Layer 2) of the OSI Model.

Think of it like a device‚Äôs fingerprint. Every network-enabled device (laptop, phone, router, printer, CCTV, etc.) has its own MAC address.

How it looks (Format)

  • MAC addresses are written in hexadecimal format and usually appear as:
  • AA:BB:CC:DD:EE:FF
  • or
  • AA-BB-CC-DD-EE-FF

Example:

  • A4:5E:60:9B:3F:21

Key Characteristics

  • Unique for each device
  • Assigned permanently by the manufacturer
  • Used inside Local Networks (LAN)
  • Helps switches identify devices
  • Generally does not travel over the internet
  • Can be spoofed (but originally fixed)

Where it is used

  • Device identification in LAN
  • Network security filtering (MAC filtering)
  • Communication inside local networks
  • Switch forwarding decisions

8. Networking Security Devices

Networking security devices are hardware or software tools that protect an organization‚Äôs network from cyber-attacks, unauthorized access, and malicious traffic. They help maintain Confidentiality, Integrity, and Availability (CIA) of network resources and are critical for SOC operations.

1. Firewall

Definition:

A firewall is a security device or software that monitors, filters, and controls network traffic between trusted internal networks and untrusted external networks (like the internet) based on a set of security rules.

Key Functions:

  • Blocks unauthorized access
  • Allows legitimate traffic only
  • Protects internal network from hackers and malware
  • Can be hardware-based or software-based

Types:

  • Network Firewalls
  • Next-Generation Firewalls (NGFW)
  • Stateful and Stateless Firewalls

SOC Relevance:

  • Monitored for blocked traffic, port scanning, or unusual access patterns
  • Generates alerts for suspicious activity

2. VPN (Virtual Private Network)

Definition:

A VPN establishes a secure and encrypted communication tunnel over public networks, allowing remote users to safely access internal network resources.

Key Functions:

  • Encrypts data traffic to prevent interception
  • Masks real IP address
  • Provides secure access for remote employees or branches

SOC Relevance:

  • VPN logs can be analyzed for unauthorized access attempts
  • Detect anomalies like unusual login locations or times

3. IDS (Intrusion Detection System)

Definition:

An IDS monitors network traffic for signs of suspicious or malicious activity and generates alerts.

Key Functions:

  • Detects unauthorized access attempts, malware, or abnormal behavior
  • Passive system ‚Äì does not block traffic automatically
  • Can be network-based (NIDS) or host-based (HIDS)

SOC Relevance:

  • Used for threat detection and alert generation
  • Provides logs for incident analysis

4. IPS (Intrusion Prevention System)

Definition:

An IPS not only detects suspicious activities like an IDS but also actively prevents attacks by blocking malicious traffic in real-time.

Key Functions:

  • Real-time attack prevention
  • Can block malicious IP addresses or ports
  • Reduces risk of data breaches or service disruption

SOC Relevance:

  • Monitored in SOC for active attack prevention
  • Critical for automated response in high-risk environments

5. DMZ (Demilitarized Zone)

Definition:

A DMZ is a separate, isolated network segment that hosts public-facing services such as web servers, email servers, or FTP servers. It acts as a buffer between the internet and internal networks.

Key Functions:

  • Protects internal network from direct external attacks
  • Hosts services that must be accessible from the internet
  • Can include multiple layers of firewalls for added security

SOC Relevance:

  • Monitored for intrusion attempts
  • Logs help detect reconnaissance, web attacks, or malware targeting public servers

6. Additional Security Devices

  • UTM (Unified Threat Management): Combines multiple security features like firewall, IDS/IPS, antivirus, and VPN into a single device.
  • Proxy Server: Acts as an intermediary for client requests, filtering traffic, caching data, and providing anonymity.
  • Network Access Control (NAC): Ensures that only authorized and compliant devices can access the network.
  • Load Balancer with Security: Distributes traffic while mitigating DDoS attacks.

9. DNS (Domain Name System)

The Domain Name System (DNS) is a critical component of networking that translates human-readable domain names into machine-readable IP addresses. It acts like the phonebook of the internet, allowing users to access websites and services without remembering numerical IP addresses.

Why DNS is Important

  • Computers and servers communicate using IP addresses.
  • Humans prefer easy-to-remember names (e.g., google.com instead of 142.250.183.110).
  • DNS automatically converts these names to the corresponding IP address so the browser or application can connect to the correct server.

Example:

Domain Name: google.com
DNS translates it to: 142.250.183.110

Without DNS, users would need to remember numeric IP addresses for every website or service.

How DNS Works (Step by Step)

  1. A user types a domain name (e.g., example.com) into a browser.
  2. The request goes to a DNS Resolver (usually provided by ISP).
  3. The DNS Resolver checks if it has the IP cached.
    • If yes ‚Üí returns the IP immediately.
    • If no ‚Üí queries other DNS servers (root server ‚Üí TLD server ‚Üí authoritative server).
  4. Once the IP address is found, it is sent back to the user‚Äôs device.
  5. The device uses this IP to connect to the web server and fetch the requested data.

Types of DNS Records

Record Type Purpose
A Record Maps domain to IPv4 address
AAAA Record Maps domain to IPv6 address
CNAME Alias for another domain
MX Mail exchange server (for emails)
PTR Reverse DNS lookup (IP ‚Üí Domain)
NS Name servers for the domain
TXT Text records (used for SPF, DKIM, verification)

DNS Security in SOC

DNS is often targeted by attackers. SOC teams monitor DNS for threats such as:

  • DNS Spoofing / Cache Poisoning: Attackers redirect traffic to malicious IPs.
  • Domain Generation Algorithms (DGA): Used by malware to dynamically create domains for command-and-control.
  • DNS Tunneling: Hiding malicious data or exfiltration within DNS queries.
  • Unauthorized changes in MX/NS records for phishing attacks.

SOC analysts use DNS logs and monitoring tools to detect, investigate, and respond to these threats.

Key Takeaways

  • DNS = ‚ÄúInternet phonebook‚Äù (Domain ‚Üí IP)
  • Allows human-friendly domain names
  • Vital for web browsing, email, and all internet services
  • SOC relevance: monitor DNS for suspicious activity to prevent attacks

10. Networking Protocols

Definition:
A network protocol is a set of standardized rules that define how data is formatted, transmitted, received, and interpreted across networks. Protocols ensure that devices with different hardware and software can communicate reliably. Think of protocols as the language that devices use to understand each other.

Common Networking Protocols

1. HTTP (Hypertext Transfer Protocol)

  • Used for web browsing
  • Transfers web pages from server to browser
  • Not secure (data is sent in plaintext)
  • Port: 80

Example: Accessing http://example.com

2. HTTPS (Hypertext Transfer Protocol Secure)

  • Secure version of HTTP
  • Uses SSL/TLS encryption to protect data during transmission
  • Ensures confidentiality and integrity
  • Port: 443

Example: Accessing https://google.com

3. FTP (File Transfer Protocol)

  • Used to transfer files between client and server
  • Supports uploading and downloading files
  • Not encrypted by default (can use FTPS for security)
  • Ports: 20 (data), 21 (control)

Example: Uploading website files to a hosting server

4. SMTP (Simple Mail Transfer Protocol)

  • Used for sending emails
  • Works between mail servers and client applications
  • Port: 25 (default), 587 (submission)

5. POP3 / IMAP (Post Office Protocol / Internet Message Access Protocol)

  • POP3: Downloads emails from server to local device (deletes from server usually)
  • IMAP: Accesses emails on the server without deleting them, allows syncing across devices
  • Ports: POP3 ‚Üí 110, IMAP ‚Üí 143 (encrypted versions use 995/993)

6. Telnet

  • Used for remote login to devices
  • Sends commands to a remote system
  • Insecure: Data, including passwords, is transmitted in plaintext
  • Port: 23

Note: Telnet is mostly replaced by SSH in modern networks.

7. SSH (Secure Shell)

  • Secure version of Telnet
  • Encrypts communication between client and server
  • Used for remote administration of servers
  • Port: 22

SOC Relevance:

  • SSH logs are monitored for unauthorized login attempts
  • Failed login attempts may indicate brute-force attacks

8. SSL / TLS (Secure Sockets Layer / Transport Layer Security)

  • Provides encryption and secure communication over networks
  • TLS is the modern version replacing SSL
  • Used by HTTPS, email security, VPNs, and other applications

SOC Relevance:

  • Ensures data confidentiality and integrity
  • Logs help detect suspicious certificate issues or misconfigurations

9. RDP (Remote Desktop Protocol)

  • Used for remote desktop access to Windows machines
  • Port: 3389
  • Allows full graphical access to a remote computer
  • Must be secured (VPN + strong authentication)

SOC Relevance:

  • Monitored for unauthorized remote access attempts
  • Brute-force or unusual RDP access is a common attack vector

Key Points

  • Protocols define how data moves, how devices talk, and how secure communication occurs
  • Each protocol has a specific purpose and port
  • SOC teams monitor protocol usage to detect unusual or malicious activity
  • Securing protocols (HTTPS, SSH, TLS, VPN) is critical for protecting sensitive data
