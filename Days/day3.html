1. What is Networking

Definition: A computer network is a collection of devices connected to share resources, communicate, and exchange data securely.

SOC Relevance

  • Monitoring network traffic helps detect malware, unauthorized access, and abnormal behavior.
  • Understanding network layout is critical for incident response and threat hunting.

Components

  • Hardware Networking: Routers, switches, hubs, access points, cabling. SOC uses these for traffic monitoring, segmentation, and device identification.
  • Software Networking: Protocols, network OS, and management tools. SOC monitors logs, protocol usage, and anomalies.

2. Networking Types (SOC Focus)

Type Range Technology Use Case SOC Monitoring Points
PAN <10m Bluetooth, USB Wearables, phones Detect rogue devices, unauthorized pairing
LAN Up to 2 km Ethernet, Wi-Fi Offices, schools Monitor internal traffic, rogue devices, lateral movement
MAN 5â€“50 km Fiber, copper City-wide networks Track inter-LAN traffic, suspicious connections, DDoS patterns

Key SOC Takeaways

  • Internal vs external traffic distinction is crucial.
  • IP address monitoring helps detect internal attacks, lateral movement, and compromised hosts.
  • Segmentation (LAN, DMZ, VLAN) reduces attack impact.

3. Networking Topologies (SOC Perspective)

Definition: Layout of devices in a network. SOC relevance is monitoring points and attack vectors.

Topology Description SOC Relevance
Bus Single backbone; simple One failure can affect traffic; SOC monitors collisions, unauthorized taps
Star Central hub/switch Central point monitoring; easier to detect rogue devices
Ring Circular SOC monitors token passing, failure points, unusual traffic loops
Mesh Fully connected High redundancy; SOC focuses on routing anomalies and traffic anomalies
Hybrid Combination Complex, SOC maps for incident response and segmentation

4. Networking Devices (SOC Deep Focus)

Device Function SOC Relevance
Switch Connect LAN devices using MAC addresses Monitor MAC changes, rogue devices, ARP spoofing
Router Connects networks; forwards packets by IP Monitor routing anomalies, external attacks, port scans
Hub Broadcast traffic Rarely used; SOC monitors sniffing risk
Repeater Amplify signals Ensure no signal degradation affects monitoring
Access Point Wi-Fi connectivity Detect rogue APs, unauthorized connections, Wi-Fi sniffing

SOC Notes

  • VLANs and segmentation are critical to limit lateral movement.
  • Monitoring MAC/IP logs helps detect unauthorized devices.
  • SOC analysts use NetFlow, SPAN ports, and SIEM for traffic analysis.

5. IP Addresses & MAC Address

IP Addresses

  • IPv4: 32-bit, limited, example: 192.168.1.10
  • IPv6: 128-bit, vast, example: 2001:db8:85a3::8a2e

Public IP

Global, ISP-assigned, monitors external attacks, reconnaissance, and data exfiltration.

Private IP

Internal traffic; SOC monitors lateral movement between hosts and unauthorized communications.

MAC Address

Hardware identifier; SOC monitors for spoofing and rogue devices.

SOC Tools

  • SIEM
  • NAC (Network Access Control)
  • Packet sniffers (Wireshark)
  • IPAM systems

6. Networking Security Devices (Deep SOC)

Device Function SOC Role
Firewall Filter traffic Alert on unusual ports, block known threats

IDS Detect attacks Alerts for scans, malware, policy violations

UTM / NAC / Proxy Unified security SOC monitors device compliance, traffic inspection, content filtering

SOC Focus

  • Configure logging and alerts for all devices.
  • Correlate events across devices in SIEM for early detection.

7. DNS (Domain Name System)

Definition: Converts human-readable domains â†’ IP addresses.

SOC Relevance: Monitoring for DNS-based attacks.

Threats Monitored

  • DNS spoofing / cache poisoning
  • DNS tunneling (malware data exfiltration)
  • Malicious domains (DGA malware detection)

SOC Actions

  • Monitor logs for unusual query patterns
  • Block or alert on known malicious domains
  • Track anomalous external DNS activity

DNS Records Monitored

A, AAAA, MX, CNAME, PTR, NS, TXT

10. Incident Response Life Cycle

  1. Preparation: Policies, tools, and monitoring setup.
  2. Identification: Detect anomalies via SIEM/IDS/IPS.
  3. Containment: Isolate compromised systems to prevent spread.
  4. Eradication: Remove malware, patch vulnerabilities.
  5. Recovery: Restore systems; continuous monitoring.
  6. Lessons Learned: Update playbooks, improve detection.

SOC Tools: SIEM, EDR, ticketing systems, forensic tools.

12. Additional SOC Networking Focus Areas

  • VLANs & Segmentation: Limit lateral movement.
  • NetFlow / Packet Capture: Detect abnormal flows, lateral movement, malware.
  • Port & Protocol Monitoring: Identify unusual access, brute-force attempts.
  • Threat Intelligence Integration: Correlate IP, domain, and malware reputation.
  • Anomaly Detection: Baseline network behavior, detect deviations.
  • Logging & SIEM Correlation: Centralized monitoring for detection and investigation.
